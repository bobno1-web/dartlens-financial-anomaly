# 프로젝트 목표 (project-goal)

> **목적:** 이 도구가 **무엇을 왜** 하는지, 어디까지가 범위인지, 무엇을 주장하고 무엇을 주장하지 않는지를 한 곳에서 정의한다. (스코프 확정 사항은 `docs/DECISIONS.md`.)

## 목표
OpenDART API로 대상 회사의 **연결(CFS) 재무제표**를 수집하고, 산업으로 분류하고, 산업 벤치마크를 계산하고, 대상 회사의 **재무비율**을 동종 업계 벤치마크와 비교하여, 검토 후보(review candidates)를 Excel로 내보낸다. 모든 출력값은 원본 filing으로 역추적 가능하다.

## MVP 범위
- **연차보고서만** (`reprt_code = 11011`)
- **연결재무제표 우선** (`fs_div = CFS`)
- **단일 사업연도** (run 당 하나의 `bsns_year`)
- **캐시 우선(cache-first)** 수집 — 재실행은 재현 가능하며 API를 다시 호출하지 않는다
- 동종업계 = **같은 유효 KSIC prefix의 상장사**
- 출력 = 타임스탬프가 붙은 단일 `.xlsx` 리포트

## 이 도구가 주장하는 것
- 플래그된 항목은 **동종업계 대비 통계적 이상치**다 — 곧 "사람이 한 번 검토해야 할 후보".

## 이 도구가 주장하지 않는 것
- 플래그는 **부정·오류·왜곡표시(fraud/error/misstatement)의 결론이 아니다.**
- `NORMAL` 라벨이 회사의 건전성·정확성을 **보증하지 않는다.**
- 벤치마크는 동종업계 표본 수와 데이터 품질에 의존한다. 표본이 적거나 분산이 작으면 그렇게 라벨링되며(`INSUFFICIENT_PEERS`, `INSUFFICIENT_VARIANCE`) 주의해서 해석해야 한다.

## 비범위 (지금은 하지 않음)
- 웹앱 · 대시보드 · 실시간 서비스
- 다년도 · 분기 비교 (향후 확장)
- 자동 판정 · 결론 생성 (도구는 후보만 제시, 판단은 사람이)
